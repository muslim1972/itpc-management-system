# استراتيجية نشر نظام إدارة ITPC (Deployment Strategy)

هذا المستند يوضح الخطوات الدقيقة لنقل النظام من بيئة التطوير المحلية إلى السحاب (Cloud) ليعمل كـ PWA مع قاعدة بيانات PostgreSQL ومزامنة لحظية.

## 1. الأهداف الأساسية
*   تشغيل النظام 24/7 برابط عام آمن (HTTPS).
*   دعم وصول 8-10 موظفين في آن واحد.
*   تفعيل ميزات PWA (التثبيت على الهاتف، العمل بمرونة).
*   مزامنة البيانات لحظياً بين جميع المستخدمين.

---

## 2. خارطة الطريق (Roadmap)

### المرحلة الأولى: تهيئة المشروع (تبدأ الآن)
*   **Git Init:** تحويل المجلد الحالي إلى مستودع Git.
*   **Gitignore:** استبعاد ملفات `itpc.db` و `node_modules` لضمان أمن ونظافة الكود.
*   **GitHub:** رفع الكود إلى مستودع خاص (Private Repository) باسم `itpc-network`.

### المرحلة الثانية: البرمجة للإنتاج (Production Ready)
*   **الخلفية (Backend):** تعديل `database.py` ليدعم متغير البيئة `DATABASE_URL`.
*   **المكتبات:** إضافة `psycopg2-binary` و `gunicorn` لملف `requirements.txt`.
*   **التشغيل:** إنشاء ملف `Procfile` لإخبار السيرفر بآلية التشغيل.

### المرحلة الثالثة: البنية التحتية (Render)
*   إنشاء قاعدة بيانات **PostgreSQL** (Managed Database).
*   إنشاء **Web Service** وربطه بمستودع GitHub.
*   إضافة **Static Site** لاستضافة واجهة React (أو دمجها مع Flask في خدمة واحدة لتقليل الجهد).

### المرحلة الرابعة: الهجرة (Migration)
*   تشغيل سكربت `migrate.py` لنقل البيانات من الملف المحلي `itpc.db` إلى قاعدة البيانات الجديدة في السحاب.

---

## 3. الخطوة الأولى المطلوبة منك الآن
من فضلك، قم بتنفيذ الأوامر التالية في الطرفية (Terminal) لتهيئة الكود للنشر:

```powershell
# 1. تهيئة المستودع
git init

# 2. إنشاء ملف الاستثناءات
echo "node_modules/
dist/
__pycache__/
*.pyc
.env
src/itpc.db" > .gitignore

# 3. إضافة الملفات وعمل أول توثيق
git add .
git commit -m "Prepare project for PWA deployment"
```

بعد التنفيذ، اذهب إلى GitHub وأنشئ مستودعاً جديداً باسم `itpc-network` ثم أخبرني بالرابط لنقوم بالربط.
